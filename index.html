Architecture

Multi-layer parallel state engine

Unlike traditional linear state machines, Parallel Path lets you define multiple independent workflow layers that run in parallel within a single instance. Layers communicate through domain events, not direct coupling.

layers

Independent layer execution

Each layer has its own states, transitions, and guards — all within one workflow instance.

sync_alt

Event-driven coordination

Layers communicate via domain events. No direct method calls, no tight coupling.

verified

Guarded autonomy

Every transition passes through a configurable guard registry before any state change occurs.
